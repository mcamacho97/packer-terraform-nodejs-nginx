## Overview

This project demonstrates the use of HashiCorp Packer and Terraform to automate the creation and deployment of a Node.js application running behind Nginx on AWS.

The solution follows Infrastructure as Code (IaC) principles and immutable infrastructure practices by creating a custom Amazon Machine Image (AMI) that contains:

* Ubuntu Server 24.04 LTS
* Node.js
* Nginx
* Node.js sample application
* Systemd service configuration
* Nginx reverse proxy configuration

The generated AMI is then consumed by Terraform to provision an EC2 instance automatically.

## Packer Workflow

Packer performs the following actions:

1. Launches a temporary EC2 instance.
2. Connects through SSH.
3. Installs Node.js.
4. Installs Nginx.
5. Copies the application files.
6. Configures a systemd service.
7. Configures Nginx as a reverse proxy.
8. Executes validation scripts.
9. Creates a reusable AMI.
10. Removes temporary resources.

### Build Command

```bash
packer init templates/aws.pkr.hcl

packer validate templates/aws.pkr.hcl

packer build templates/aws.pkr.hcl
```

---

## Terraform Workflow

Terraform consumes the AMI generated by Packer through the generated `manifest.json` file and provisions:

* Security Group
* EC2 Instance
* Public IP Output

### Deploy Command

```bash
terraform init

terraform plan

terraform apply
```

## Multi-Cloud Support

The project includes an additional Azure builder template (`azure.pkr.hcl`) to demonstrate portability and multi-cloud capabilities.

This approach allows the same provisioning scripts and application configuration to be reused across different cloud providers, reducing vendor lock-in and increasing deployment flexibility.
